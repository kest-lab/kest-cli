# 🎯 KEST 特色端口设计

## 端口号: **5119**

### 🎨 设计理念

KEST 使用 **5119** 作为默认端口，这个数字来源于 KEST 品牌名称的字母序号：

```
K = 11 (第11个字母)
E = 5  (第5个字母)  ✓
S = 19 (第19个字母) ✓
T = 20 (第20个字母)

5119 = E(5) + S(19)
```

### ✨ 优势

1. **品牌特色** 🏷️
   - 独一无二的端口号
   - 体现 KEST 品牌标识
   - 易于识别和记忆

2. **避免冲突** 🚫
   - 不是常见的 8080、3000、8000 等
   - 减少与其他服务的端口冲突
   - 适合开源项目独立运行

3. **技术合理** ✅
   - 在有效端口范围内 (1024-65535)
   - 无需 root 权限 (>1024)
   - 符合网络标准

4. **易于记忆** 🧠
   - 51-19 两组数字
   - 与 KEST 品牌关联
   - 开发者友好

### 📝 使用说明

#### 本地开发

```bash
# 默认使用 5119 端口
go run cmd/api/main.go

# 访问 API
curl http://localhost:5119/v1/health
```

#### Docker 部署

```bash
# 使用默认端口 5119
docker run -p 5119:5119 kest-api

# 自定义端口
docker run -p 8080:5119 -e PORT=8080 kest-api
```

#### 云平台部署

云平台（Render、Zeabur、Railway）会自动注入 `PORT` 环境变量，无需手动配置。

### 🔧 配置

#### 环境变量

```bash
# 使用默认端口 5119
SERVER_PORT=5119

# 或使用自定义端口
SERVER_PORT=8080
```

#### 配置文件

```yaml
# .kest/config.yaml
environments:
  local:
    base_url: http://127.0.0.1:5119
```

### 🌐 端口映射

| 环境 | 默认端口 | 说明 |
|------|---------|------|
| 本地开发 | 5119 | KEST 特色端口 |
| Docker | 5119 | 容器内部端口 |
| 云平台 | 动态 | 由平台自动分配 |

### 💡 为什么不用其他端口？

| 端口 | 问题 |
|------|------|
| 8080 | 太常见，容易冲突 |
| 3000 | Node.js 默认，冲突率高 |
| 8000 | Python 默认，较常见 |
| 7111 | 无特殊含义，不易记 |
| 8025 | 无品牌关联 |

**5119** 不仅仅是一个端口号，它是 KEST 品牌的一部分：
- 🎯 独特性：避免与其他服务冲突
- 🏷️ 品牌化：体现 KEST 特色
- 🧠 易记性：51-19，简单好记
- ✅ 合理性：符合技术标准

# 速率限制

当前后端代码里还没有稳定、公开的产品级限流契约，因此本版文档站不声明具体套餐额度。

## 建议默认假设

- 客户端应能处理 `429 Too Many Requests`
- 对同步和轮询类流程应使用退避重试
- 不要让多个并行 worker 高频写入同一个项目

## 推荐客户端行为

- 使用指数退避
- 对 CLI Sync 流程尽量做幂等设计
- 尽量批量上传 spec，而不是每个端点发一条请求

后续如果后端公开稳定的 rate-limit header 或额度规则，这一页应与契约同步更新。

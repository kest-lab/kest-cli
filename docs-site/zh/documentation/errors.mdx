# 错误处理

Kest 的大多数接口都使用统一 JSON 包络。

## 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

## 错误响应

```json
{
  "code": 400,
  "message": "Invalid request data",
  "error": "validation details"
}
```

## 常见状态码

- `200` 正常读取或更新成功
- `201` 创建成功
- `204` 删除成功且无响应体
- `400` 参数或请求体不合法
- `401` 未认证或认证失效
- `403` 项目或工作区权限不足
- `404` 资源不存在
- `409` 资源冲突，例如 slug 重复
- `503` 某个子系统暂时不可用，例如 CLI syncer 未配置

## 关于列表响应

有些列表接口返回 `data.items` 和自定义 `meta`，有些则使用共享响应包里的分页结构。某个具体接口以 `API 参考` 中的 schema 为准。

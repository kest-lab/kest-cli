# API Spec 生命周期

Kest 把 API 规格视为持续维护的工作资产，而不是一次性上传文件。

## 典型流程

1. 在本地编写或发现接口
2. 通过 CLI Sync 或平台 API 把它们写入项目
3. 继续补充 summary、description、examples 和生成文档
4. 从平台导出规格或基于规格生成测试用例

## 关联 API 区域

- Projects
- CLI Sync
- API Specs
- Test Cases

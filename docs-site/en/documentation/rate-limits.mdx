# Rate limits

Kest does not currently publish a stable, product-level rate limit contract in backend code. Because of that, this first Mintlify release does not claim exact per-tier quotas.

## What to assume

- Write clients to tolerate `429 Too Many Requests`.
- Add retries with backoff for sync and polling workflows.
- Avoid blasting project-scoped write endpoints from parallel workers.

## Recommended client behavior

- Use exponential backoff for retries.
- Make CLI sync idempotent on the client where practical.
- Batch spec uploads instead of sending one request per endpoint.

When Kest starts emitting stable rate-limit headers or public quotas, this page should be updated alongside the backend contract.

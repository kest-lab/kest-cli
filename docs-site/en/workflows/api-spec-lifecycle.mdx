# API spec lifecycle

Kest treats API specs as a working artifact rather than a static upload.

## Typical lifecycle

1. Author or discover endpoints locally.
2. Sync them into a project through CLI sync or create them manually through the platform API.
3. Refine summaries, descriptions, examples, and generated docs.
4. Export the managed spec or derive test cases from it.

## Related API areas

- Projects
- CLI Sync
- API Specs
- Test Cases

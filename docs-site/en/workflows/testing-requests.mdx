# Testing requests

A common Kest workflow looks like this:

1. Create a project.
2. Add collections and requests.
3. Model environments and variables.
4. Save or sync API specs.
5. Generate or author test cases.
6. Run test cases and inspect run history.

The reference sections for `Collections`, `Requests`, `Environments`, and `Test Cases` map directly to that workflow.

# Project collaboration

Kest currently exposes two collaboration layers:

- `Projects` with member roles such as `owner`, `admin`, `write`, and `read`
- `Workspaces` with roles such as `owner`, `admin`, `editor`, and `viewer`

Use project membership for API assets and execution access. Use workspaces when you need a broader container for grouped collaboration.

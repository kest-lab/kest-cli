# Kest documentation

Kest is an AI-native API testing platform built around a terminal-first CLI and a collaborative web console.

This site is organized in two layers:

- `Documentation` explains how to work with authentication, error handling, CLI sync, and common workflows.
- `API Reference` is generated from Kest's public OpenAPI contract and is the source of truth for endpoints, request bodies, and response schemas.

## What you can do with Kest

- Create and organize API projects.
- Import or sync API specs from local CLI workflows.
- Model environments, collections, requests, and reusable examples.
- Generate, run, and inspect test cases.
- Collaborate with project members using scoped roles.

## Coverage in this first Mintlify version

This documentation site intentionally follows the real API surface exposed by the current backend codebase. It covers:

- Authentication and user endpoints
- Projects and CLI sync
- API specs
- Environments
- Categories
- Test cases
- Members and permissions
- Collections, requests, and history
- Workspaces
- Health and system metadata

Legacy Markdown under `/docs/api` remains useful as source material, but this site prefers the current code routes and the new OpenAPI files in `/docs-site/openapi`.
